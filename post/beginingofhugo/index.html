BeginingOfHugo

Feb 18, 2020
1 min read

windows系统的用户去 hugo二进制文件和tar包, 如果你的电脑是64位的话,推荐extends那个，记得那个比普通版的hugo支持将sass和less转为css。

然后把环境变量设置在解压出来的hugo.exe目录下。

mac用户直接在你要新建blog项目的目录下

$ brew install hugo

linux用户用adt-get，yum等，根据不同系统不同选择进行下载。

Quick Start

Create a new post

$ hugo new site myBlog

此时会自动在当前目录下生成myBlog的文件,里面有各种各样的文件， 如 archetypes, content, layouts, static, config.toml

Download a theme

你可以到 hugo主题 查看主题,再把对应主题git clone下来即可。 比如下面这个例子,我在根目录下输入下面的命令下载m10c的主题。

$ git clone https://github.com/vaga/hugo-theme-m10c.git themes/m10c

按回车后会自动把m10c下载到theme目录下。

Run server

$ hugo server -t m10c --buildDrafts

此时会默认在 http://localhost:1313/ 开启服务器，你可以在浏览器输入这个地址访问查看效果。 这里-t指定主题，此时我用的是m10c的主题, –buildDrafts是生成屋的意思, 不必纠结。

Create a new md

新建一篇文章，比如这里我在post目录下新建一篇叫hello.md的Markdown格式文章。

$ hugo new post/hello.md

然后在content/post就可以找到hello.md挥洒你的笔墨了。

Generate static resource

$ hugo --theme="m10c" --baseUrl="https://chen_shu_ming.gitee.io/myblogu/" --buildDrafts

我这里主题是m10c,baseUrl一项填你打算部署的域名,U注意大写，–buildDrafts不要拉下。 执行成功后，会在根目录下生成public文件夹，里面是静态资源html,css,js文件等。

进入public目录

$ cd public

一波git操作。

$ git add .
$ git commit -m "提交附加信息"
$ git remote add origin 【这里是远程仓库地址,如https://xxxxx.git】
$ git push -u origin master

刷新后即可，如果是选择gitee部署，记得在gitpage那部署。 如果是在github上记得仓库命名有要求。
